Компонент всегда представляет собой директорию. В директории всегда есть файл самого компонента (называется так же, как директория), файл index.ts, который реэкспортирует компонент.
Остальные файлы опциональны.

Если внутри этого компонента есть отдельные компоненты, они тоже представляют собой директории и оформляются по тем же правилам. 

Интерфейс props, который передаются в компонент всегда называется IProps, для каждого свойства интерфейса должен быть описан комментарий.
Тип определяется не для объекта пропсов, а для всего компонента, пример: `... const Component: React.FC<IProps> = (props) => ...`.
Все остальные модели, используемые в компоненте кладутся в файл models.ts в этой же директории.
props всегда деструктуризуются в первой строке компонента, но не в параметрах функции компонента.

Если в компоненте используются константы, то они оформляются в uppercase, соединяя слова через нижнее подчеркивание.
Эти константы кладутся в файл consts.ts.

Если в компоненте используются утилиты, которые можно выделить отдельно от компонента, то создается директория utils.
Внутри нее для каждой утилиты создаются файлы по называнию утилит, а также файл index.ts, который реэкспортирует все утилиты.
